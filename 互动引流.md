# 互动引流工作流

## 目标

通过搜索和互动，提升账号在相关话题下的活跃度与粘性：
- 增加账号曝光与互动量
- 借由高质量评论，让账号与目标话题建立更强关联

## 前置条件（必须全部完成）

```
□ 已加载 用户配置.md(./用户配置.md)
□ 已加载 全局规则.md(./全局规则.md)（含评论筛选过滤规则）
□ Bridge Server 在线：bash ~/.claude/skills/xiaohongshu-cli/scripts/bridge.sh ensure
□ 油猴脚本已连接：node cli.js status
□ REPLIED_CIDS 集合已初始化（使用全局规则.md中的执行流程）
```

## 🔴 核心安全规则（必须严格遵守）

### 规则 1：命令串行 + 强制随机间隔

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 严禁并发执行任何 CLI 命令                                  │
│  ❌ 严禁使用固定间隔（必须每次重新随机）                         │
│  ❌ 严禁跳过 sleep（即使"感觉"不需要）                          │
│                                                              │
│  ✅ 所有命令必须逐条串行执行                                   │
│  ✅ 每条命令完成后必须等待 40-55 秒随机间隔                      │
│  ✅ 每次等待前必须重新生成随机数                                │
└─────────────────────────────────────────────────────────────┘
```

**强制执行模板**（复制使用，不要修改结构）：

```bash
# ═══ 执行单条命令的模板 ═══
execute_with_delay() {
  local cmd="$1"
  local delay=$((40 + RANDOM % 16))  # 40-55 秒随机值
  
  echo "▶ 执行: $cmd"
  eval "$cmd"
  
  if [ $? -eq 0 ]; then
    echo "✓ 成功，等待 ${delay} 秒..."
    sleep "$delay"
  else
    echo "✗ 失败，跳过等待"
  fi
}
```

### 规则 2：不重复回复

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 同一条评论 cid 一生只能被回复一次（跨日跨轮均生效）           │
│                                                              │
│  ✅ 每次回复前必须检查 cid 是否在 REPLIED_CIDS 中              │
│  ✅ 回复成功后必须将 cid 加入 REPLIED_CIDS                    │
└─────────────────────────────────────────────────────────────┘
```

**强制执行流程**：

```bash
# ═══ 检查是否已回复 ═══
is_replied() {
  local cid="$1"
  grep -q "$cid" /tmp/replied_cids.txt 2>/dev/null
  return $?
}

# ═══ 回复前检查 ═══
if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
else
  execute_with_delay 'node cli.js post <note_id> "内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
fi
```

## 执行流程

### Step 1: 获取现有笔记数据

先获取自己已发布的笔记，了解当前内容方向：

```bash
# ═══ 获取我的笔记列表（使用执行模板）═══
execute_with_delay 'node cli.js my'
```

**分析要点**：
- 记录笔记的主要话题/领域（如 AI、编程、Agent 等）
- 对于标题信息不足的笔记，获取详情补充了解：
  ```bash
  execute_with_delay 'node cli.js note <note_id>'
  ```

### Step 2: 根据笔记内容生成赛道关键词并搜索

根据 Step 1 分析的笔记内容，生成相关搜索关键词：

**关键词生成规则**：
- 从笔记标题/描述中提取核心话题
- 扩展相关术语（如：AI → 人工智能、机器学习、深度学习）
- 考虑当前热点（如：新发布的 AI 产品、热门框架）

**执行搜索**（每个关键词搜索一次，使用执行模板）：

```bash
# ═══ 搜索相关关键词（使用执行模板）═══
execute_with_delay 'node cli.js search "AI Agent" --count 20'
execute_with_delay 'node cli.js search "Claude" --count 20'
execute_with_delay 'node cli.js search "智能体" --count 20'
execute_with_delay 'node cli.js search "大模型" --count 20'
```

**搜索策略**：
- 每轮选择 3-5 个关键词
- 优先搜索与自身内容相关的关键词
- 适当搜索热点话题增加曝光

### Step 3: 筛选合适的笔记

对搜索结果进行筛选，找到值得互动的笔记：

**筛选标准**：
```
✓ 发布时间：优先 24 小时内，最久不超过 7 天
✓ 评论数量：≥ 5 条（有讨论热度）
✓ 内容相关：与自身定位（AI/技术/编程）相关
✓ 排除内容：纯广告、营销号、竞品内容
```

**获取评论详情**（使用执行模板，必须使用 `--page 1` 限制）：

```bash
# ═══ 获取笔记评论（使用执行模板，必须使用 --page 1 限制）═══
execute_with_delay 'node cli.js get <note_id> --page 1 --depth 1'
```

**⚠️ 重要：获取评论必须使用 `--page 1` 限制**
- ❌ 错误：`node cli.js get <note_id> --all --depth 1`（获取所有评论，容易触发风控）
- ✅ 正确：`node cli.js get <note_id> --page 1 --depth 1`（只获取第一页，安全）

**评论筛选**（遵循 `全局规则.md` 第 4 节）：
- 跳过：祈愿/接龙、简短感叹、求购/求渠道、祝福/客套
- 优先：笔记作者评论、有深度提问、讨论性评论、分享经验

### Step 4: 生成评论内容

1. 加载 `评论风格指南.md(./评论风格指南.md)`
2. 根据评论内容选择合适的风格（每轮至少用 3 种）
3. 生成回复内容（确保不违反 `全局规则.md` 的内容禁令）

**生成要点**：
- 有实质内容，能引发讨论
- 体现专业度或提供价值
- 符合小红书社区调性

### Step 5: 执行评论或回复

**发表评论**（对笔记本身）：

```bash
# ═══ 对笔记发表评论（使用执行模板）═══
execute_with_delay 'node cli.js post <note_id> "评论内容"'
```

**回复评论**（对其他用户的评论）：

```bash
# ═══ 回复特定评论（使用执行模板 + REPLIED_CIDS 检查）═══
if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
else
  execute_with_delay 'node cli.js post <note_id> "回复内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
fi
```

**执行要点**：
- 每条命令间隔 40-55 秒随机值（使用执行模板自动处理）
- 执行前检查 cid 是否在 REPLIED_CIDS 中
- 发布后将 cid 加入 REPLIED_CIDS

## 增量执行

后续执行时使用增量模式：

```bash
# ═══ 只获取新评论（使用执行模板，必须使用 --page 1 限制）═══
execute_with_delay 'node cli.js get <note_id> --page 1 --new --depth 1'
```

**增量策略**：
- 搜索关键词可复用（根据热点适当调整）
- 笔记筛选优先最新内容
- 评论去重依赖 REPLIED_CIDS

## 注意事项

1. **频率控制**：宁可少做，不可触发风控（使用执行模板自动处理）
2. **内容质量**：每条评论都要有实质内容
3. **风格变化**：每轮至少用 3 种不同风格
4. **去重检查**：严格执行 REPLIED_CIDS 检查
5. **评论筛选**：遵循 `全局规则.md` 第 4 节的筛选规则
6. **间隔执行**：所有命令必须使用 `execute_with_delay` 模板
7. **获取评论限制**：必须使用 `--page 1` 限制，避免触发风控

## 执行检查清单

**每执行 5 条命令后，必须检查**：

```
□ 是否所有命令都是串行执行的？
□ 每条命令之间是否等待了 40-55 秒？
□ 是否检查了 REPLIED_CIDS？
□ 是否使用了 ≥ 3 种不同风格？
□ 是否违反了内容禁令？
□ 获取评论时是否使用了 --page 1 限制？
```

**如果发现违规**：立即停止本轮，记录原因到执行报告。
