# 执行模板

> 本文件包含所有必须使用的执行模板和函数。**直接复制使用，不要修改结构。**

## 🔴 核心安全规则

### 规则 1：命令串行 + 强制随机间隔

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 严禁并发执行任何 CLI 命令                                  │
│  ❌ 严禁使用固定间隔（必须每次重新随机）                         │
│  ❌ 严禁跳过 sleep（即使"感觉"不需要）                          │
│                                                              │
│  ✅ 所有命令必须逐条串行执行                                   │
│  ✅ 每条命令完成后必须等待 40-55 秒随机间隔                      │
│  ✅ 每次等待前必须重新生成随机数                                │
└─────────────────────────────────────────────────────────────┘
```

### 规则 2：不重复回复

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 同一条评论 cid 一生只能被回复一次（跨日跨轮均生效）           │
│  ❌ 同一作者短期内不重复（≥ 7 天冷却期）                        │
│                                                              │
│  ✅ 每次回复前必须检查 cid 是否在 REPLIED_CIDS 中              │
│  ✅ 回复成功后必须将 cid 加入 REPLIED_CIDS                    │
│  ✅ 必须检查同一作者的最后互动时间（≥ 7 天）                    │
└─────────────────────────────────────────────────────────────┘
```

## 必须使用的函数

### 1. execute_with_delay - 执行单条命令（必须使用）

```bash
# ═══ 执行单条命令的模板 ═══
# 功能：执行命令并等待随机间隔
# 用法：execute_with_delay 'node cli.js search "AI Agent" --count 20'
execute_with_delay() {
  local cmd="$1"
  local delay=$((40 + RANDOM % 16))  # 40-55 秒随机值
  
  echo "▶ 执行: $cmd"
  eval "$cmd"
  
  if [ $? -eq 0 ]; then
    echo "✓ 成功，等待 ${delay} 秒..."
    sleep "$delay"
  else
    echo "✗ 失败，跳过等待"
  fi
}
```

### 2. is_replied - 检查是否已回复（必须使用）

```bash
# ═══ 检查是否已回复 ═══
# 功能：检查 cid 是否在 REPLIED_CIDS 集合中
# 用法：if is_replied "target_cid"; then echo "已回复"; fi
is_replied() {
  local cid="$1"
  grep -q "$cid" /tmp/replied_cids.txt 2>/dev/null
  return $?
}
```

### 3. check_author_cooldown - 检查作者冷却期（必须使用）

```bash
# ═══ 检查作者冷却期（≥ 7 天）═══
# 功能：检查与作者的最后互动时间是否超过 7 天
# 用法：if check_author_cooldown "user_id"; then echo "可以互动"; fi
check_author_cooldown() {
  local user_id="$1"
  local last_interaction=$(node cli.js profile "$user_id" | grep -o '"last_seen":"[^"]*"' | cut -d'"' -f4)
  
  if [ -z "$last_interaction" ]; then
    return 0  # 无记录，可以互动
  fi
  
  local last_ts=$(date -d "$last_interaction" +%s 2>/dev/null || echo 0)
  local now_ts=$(date +%s)
  local days_diff=$(( (now_ts - last_ts) / 86400 ))
  
  if [ $days_diff -ge 7 ]; then
    return 0  # 冷却期已过
  else
    echo "⏭ 跳过：作者 $user_id 冷却期未过（${days_diff}/7 天）"
    return 1  # 冷却期未过
  fi
}
```

### 4. init_replied_cids - 初始化已回复集合（必须使用）

```bash
# ═══ 初始化 REPLIED_CIDS 集合 ═══
# 功能：获取所有已回复的 cid 并保存到文件
# 用法：init_replied_cids
init_replied_cids() {
  echo "🔄 初始化 REPLIED_CIDS 集合..."
  node cli.js replied > /tmp/replied_cids.txt
  local count=$(wc -l < /tmp/replied_cids.txt)
  echo "✓ 已加载 $count 个已回复的 cid"
}
```

## 必须使用的执行流程

### 流程 1：启动前初始化

```bash
# ═══ 启动前初始化（必须执行）═══
# Step 1: 确认 Bridge Server 在线
bash ~/.claude/skills/xiaohongshu-cli/scripts/bridge.sh ensure

# Step 2: 确认油猴脚本已连接
node cli.js status

# Step 3: 初始化 REPLIED_CIDS 集合
init_replied_cids

echo "✅ 初始化完成，可以开始执行"
```

### 流程 2：执行单条命令（搜索/获取/查看）

```bash
# ═══ 执行单条命令（使用 execute_with_delay）═══
# 搜索笔记
execute_with_delay 'node cli.js search "AI Agent" --count 20'

# 获取评论（必须使用 --page 1 限制，避免触发风控）
execute_with_delay 'node cli.js get <note_id> --page 1 --depth 1'

# 查看日志
execute_with_delay 'node cli.js log --tail 20'
```

**⚠️ 重要：获取评论必须使用 `--page 1` 限制**
- ❌ 错误：`node cli.js get <note_id> --all --depth 1`（获取所有评论，容易触发风控）
- ✅ 正确：`node cli.js get <note_id> --page 1 --depth 1`（只获取第一页，安全）

### 流程 3：执行回复命令（带检查）

```bash
# ═══ 执行回复命令（带 REPLIED_CIDS 检查 + 作者冷却期检查）═══
# 目标：回复评论 cid=target_cid，作者 user_id=target_user_id

if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
elif ! check_author_cooldown "target_user_id"; then
  echo "⏭ 跳过：作者冷却期未过"
else
  execute_with_delay 'node cli.js post <note_id> "回复内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
  echo "✓ 回复成功，已记录到 REPLIED_CIDS"
fi
```

### 流程 4：批量执行多个命令

```bash
# ═══ 批量执行多个命令（使用 execute_with_delay）═══
commands=(
  'node cli.js search "AI Agent" --count 20'
  'node cli.js search "Claude" --count 20'
  'node cli.js search "智能体" --count 20'
  'node cli.js get <note_id> --all --depth 1'
  'node cli.js post <note_id> "评论内容"'
)

for cmd in "${commands[@]}"; do
  execute_with_delay "$cmd"
done
```

## 常见错误

### ❌ 错误 1：跳过 sleep

```bash
# 错误：没有等待间隔
node cli.js search "AI Agent" --count 20
node cli.js search "Claude" --count 20

# 正确：使用 execute_with_delay
execute_with_delay 'node cli.js search "AI Agent" --count 20'
execute_with_delay 'node cli.js search "Claude" --count 20'
```

### ❌ 错误 2：使用固定间隔

```bash
# 错误：使用固定间隔
sleep 45

# 正确：使用随机间隔（execute_with_delay 自动处理）
# delay=$((40 + RANDOM % 16))  # 40-55 秒随机值
```

### ❌ 错误 3：跳过 REPLIED_CIDS 检查

```bash
# 错误：直接回复
node cli.js post <note_id> "内容" --reply-to abc123

# 正确：先检查再回复
if is_replied "abc123"; then
  echo "⏭ 跳过：已回复过该评论"
else
  execute_with_delay 'node cli.js post <note_id> "内容" --reply-to abc123'
  echo "abc123" >> /tmp/replied_cids.txt
fi
```

### ❌ 错误 4：跳过作者冷却期检查

```bash
# 错误：不检查冷却期
node cli.js post <note_id> "内容" --reply-to abc123

# 正确：检查冷却期
if ! check_author_cooldown "user_id"; then
  echo "⏭ 跳过：作者冷却期未过"
else
  execute_with_delay 'node cli.js post <note_id> "内容" --reply-to abc123'
fi
```

### ❌ 错误 5：使用 --all 获取所有评论（容易触发风控）

```bash
# 错误：使用 --all 获取所有评论
node cli.js get <note_id> --all --depth 1

# 正确：使用 --page 1 只获取第一页
node cli.js get <note_id> --page 1 --depth 1
```

**为什么必须使用 `--page 1`**：
- `--all` 会获取所有评论，请求量大，容易触发小红书风控
- `--page 1` 只获取第一页（通常 20 条），足够筛选有价值的评论
- 如果需要更多评论，可以手动增加 `--page 2`、`--page 3`，但要谨慎使用

## 总结

**必须记住**：

1. **所有命令必须使用 `execute_with_delay` 模板**
2. **所有回复必须先检查 `is_replied` 和 `check_author_cooldown`**
3. **启动前必须执行 `init_replied_cids` 初始化**
4. **获取评论必须使用 `--page 1` 限制（避免触发风控）**
5. **每执行 5 条命令必须检查执行检查清单**

**违反任意一条 → 立即停止本轮，记录原因到执行报告。**
