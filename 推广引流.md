# 推广引流工作流

## 目标

针对特定话题或作者进行精准推广：
- 在目标话题下建立存在感
- 与目标作者建立互动关系
- 引导流量到自己的内容

## 前置条件（必须全部完成）

```
□ 已加载 用户配置.md
□ 已加载 全局规则.md（含评论筛选过滤规则）
□ Bridge Server 在线：bash ~/.claude/skills/xiaohongshu-cli/scripts/bridge.sh ensure
□ 油猴脚本已连接：node cli.js status
□ REPLIED_CIDS 集合已初始化（使用全局规则.md中的执行流程）
□ 已确定推广目标（话题/作者）
```

## 🔴 核心安全规则（必须严格遵守）

### 规则 1：命令串行 + 强制随机间隔

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 严禁并发执行任何 CLI 命令                                  │
│  ❌ 严禁使用固定间隔（必须每次重新随机）                         │
│  ❌ 严禁跳过 sleep（即使"感觉"不需要）                          │
│                                                              │
│  ✅ 所有命令必须逐条串行执行                                   │
│  ✅ 每条命令完成后必须等待 40-55 秒随机间隔                      │
│  ✅ 每次等待前必须重新生成随机数                                │
└─────────────────────────────────────────────────────────────┘
```

**强制执行模板**（复制使用，不要修改结构）：

```bash
# ═══ 执行单条命令的模板 ═══
execute_with_delay() {
  local cmd="$1"
  local delay=$((40 + RANDOM % 16))  # 40-55 秒随机值
  
  echo "▶ 执行: $cmd"
  eval "$cmd"
  
  if [ $? -eq 0 ]; then
    echo "✓ 成功，等待 ${delay} 秒..."
    sleep "$delay"
  else
    echo "✗ 失败，跳过等待"
  fi
}
```

### 规则 2：不重复回复

```
┌─────────────────────────────────────────────────────────────┐
│  ❌ 同一条评论 cid 一生只能被回复一次（跨日跨轮均生效）           │
│  ❌ 同一作者短期内不重复（≥ 7 天冷却期）                        │
│                                                              │
│  ✅ 每次回复前必须检查 cid 是否在 REPLIED_CIDS 中              │
│  ✅ 回复成功后必须将 cid 加入 REPLIED_CIDS                    │
│  ✅ 必须检查同一作者的最后互动时间（≥ 7 天）                    │
└─────────────────────────────────────────────────────────────┘
```

**强制执行流程**：

```bash
# ═══ 检查是否已回复 ═══
is_replied() {
  local cid="$1"
  grep -q "$cid" /tmp/replied_cids.txt 2>/dev/null
  return $?
}

# ═══ 检查作者冷却期（≥ 7 天）═══
check_author_cooldown() {
  local user_id="$1"
  local last_interaction=$(node cli.js profile "$user_id" | grep -o '"last_seen":"[^"]*"' | cut -d'"' -f4)
  
  if [ -z "$last_interaction" ]; then
    return 0  # 无记录，可以互动
  fi
  
  local last_ts=$(date -d "$last_interaction" +%s 2>/dev/null || echo 0)
  local now_ts=$(date +%s)
  local days_diff=$(( (now_ts - last_ts) / 86400 ))
  
  if [ $days_diff -ge 7 ]; then
    return 0  # 冷却期已过
  else
    echo "⏭ 跳过：作者 $user_id 冷却期未过（${days_diff}/7 天）"
    return 1  # 冷却期未过
  fi
}

# ═══ 回复前检查 ═══
if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
elif ! check_author_cooldown "target_user_id"; then
  echo "⏭ 跳过：作者冷却期未过"
else
  execute_with_delay 'node cli.js post <note_id> "内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
fi
```

## 评论筛选规则（推广场景专用）

> 推广场景比互动场景更严格，不仅筛选评论，还要筛选**笔记本身**。

### 笔记筛选标准

**跳过这些笔记**：
- 纯广告/营销内容
- 与推广目标相关度低
- 评论区冷清（< 5 条评论）
- 发布时间过久（> 30 天）

**优先选择**：
- 目标作者的笔记
- 讨论技术/方案的笔记
- 评论区有深度讨论的笔记

### 评论筛选标准

**通用规则**：遵循 `全局规则.md` 中的评论筛选过滤规则

**推广场景额外跳过**：
- 已有其他人推广过的评论区
- 评论区氛围偏负面/攻击性
- 笔记作者明显是竞品

**推广场景优先回复**：
- 笔记作者本人的评论（最高优先级）
- 提出与推广内容相关问题的评论
- 表达兴趣/需求的评论

## 执行流程

### Step 1: 确定推广目标

**话题推广**：

确定与自身内容相关的关键词：

```bash
# ═══ 搜索目标话题（使用执行模板）═══
execute_with_delay 'node cli.js search "AI Agent" --count 30'
execute_with_delay 'node cli.js search "Claude 教程" --count 30'
execute_with_delay 'node cli.js search "智能体开发" --count 30'
```

**关键词选择策略**：
- 从自己的笔记内容中提取核心话题
- 考虑目标受众可能搜索的关键词
- 适当关注当前热点话题

**作者推广**：

找到目标作者的 user_id：

```bash
# ═══ 方法 1：通过搜索笔记找到作者（使用执行模板）═══
execute_with_delay 'node cli.js search "作者昵称" --count 5'

# ═══ 方法 2：通过笔记详情获取作者信息（使用执行模板）═══
execute_with_delay 'node cli.js note <note_id>'
# 输出中会包含 author.user_id

# ═══ 方法 3：查看作者完整信息（使用执行模板）═══
execute_with_delay 'node cli.js whois <user_id>'
```

### Step 2: 筛选目标笔记

对搜索结果进行筛选：

**筛选标准**：
```
✓ 相关性：笔记内容与推广目标高度相关
✓ 发布时间：优先 7 天内，最久不超过 30 天
✓ 评论数量：≥ 5 条（有讨论空间）
✓ 内容质量：非纯广告、非竞品内容
```

**优先级排序**：
1. 自己的笔记（优先回复评论，引导其他用户）
2. 目标作者的笔记（建立互动关系）
3. 相关话题的热门笔记（扩大曝光）

**获取评论详情**（使用执行模板，必须使用 `--page 1` 限制）：

```bash
# ═══ 获取笔记评论（使用执行模板，必须使用 --page 1 限制）═══
execute_with_delay 'node cli.js get <note_id> --page 1 --depth 1'
```

**⚠️ 重要：获取评论必须使用 `--page 1` 限制**
- ❌ 错误：`node cli.js get <note_id> --all --depth 1`（获取所有评论，容易触发风控）
- ✅ 正确：`node cli.js get <note_id> --page 1 --depth 1`（只获取第一页，安全）

**评论筛选**（遵循 `全局规则.md` 第 4 节 + 推广场景额外规则）：
- 跳过：祈愿/接龙、简短感叹、求购/求渠道、祝福/客套
- 跳过：已有竞品推广、负面氛围
- 优先：笔记作者评论、相关问题、表达兴趣

### Step 3: 生成推广内容

**内容原则**：
- 自然融入，不突兀
- 提供价值（解答问题、补充信息）
- 间接引导（不直接贴链接）

**引导方式**：
```
✅ "我之前也研究过这个，写过一篇详细的，感兴趣可以看看"
✅ "这个问题我遇到过，解决方案在我的主页有分享"
✅ "这个思路很赞！补充一个相关案例，我主页有详细分析"
❌ "看我的笔记：https://..."（直接链接会被限流）
❌ "关注我获取更多"（太直接）
```

**风格选择**：
- 遵循 `评论风格指南.md` 的风格模板
- 推广场景优先使用：专业解读型、解答引导型
- 每轮至少用 2 种不同风格

### Step 4: 执行推广

**发表评论**（对笔记本身）：

```bash
# ═══ 对笔记发表评论（使用执行模板）═══
execute_with_delay 'node cli.js post <note_id> "推广内容"'
```

**回复评论**（对其他用户的评论）：

```bash
# ═══ 回复特定评论（使用执行模板 + REPLIED_CIDS 检查 + 作者冷却期检查）═══
if is_replied "target_cid"; then
  echo "⏭ 跳过：已回复过该评论"
elif ! check_author_cooldown "target_user_id"; then
  echo "⏭ 跳过：作者冷却期未过"
else
  execute_with_delay 'node cli.js post <note_id> "推广内容" --reply-to target_cid'
  echo "target_cid" >> /tmp/replied_cids.txt  # 记录已回复
fi
```

**执行要点**：
- 同一作者间隔 ≥ 7 天（使用 check_author_cooldown 函数）
- 每条命令间隔 40-55 秒随机值（使用执行模板自动处理）
- 发布后将 cid 加入 REPLIED_CIDS

- 评论去重：依赖 REPLIED_CIDS

## 推广策略详解

### 策略 A：解答引导型

**适用场景**：评论中有提出问题

**示例**：
> 评论："这个方案的性能瓶颈在哪？"
> 回复："这个问题其实可以用 [方案] 解决，我之前研究过，效果不错。关键是 [要点]。"

**要点**：
- 直接回答问题
- 提供具体方案
- 暗示有更多内容

### 策略 B：共鸣分享型

**适用场景**：评论中有分享经验或表达困惑

**示例**：
> 评论："我之前也遇到过这个问题，折腾了好久"
> 回复："深有同感！我之前也遇到过，后来发现 [经验]，分享给大家参考。"

**要点**：
- 表达共鸣
- 分享相关经验
- 提供实用建议

### 策略 C：补充延伸型

**适用场景**：评论中有讨论但可以补充更多角度

**示例**：
> 评论："这个方案不错，但成本有点高"
> 回复："说得对！补充一点，[额外信息] 这个也很重要，大家可以了解一下。"

**要点**：
- 认同原观点
- 补充有价值信息
- 引导进一步讨论

## 注意事项

1. **自然融入**：不要让推广感太强，像正常交流
2. **提供价值**：每条评论都要有实质内容
3. **遵守冷却期**：同一作者 ≥ 7 天（使用 check_author_cooldown 函数）
4. **避免竞品贴脸**：不在竞品内容下推广
5. **评论筛选**：遵循 `全局规则.md` 第 4 节 + 推广场景额外规则
6. **间隔执行**：所有命令必须使用 `execute_with_delay` 模板
7. **获取评论限制**：必须使用 `--page 1` 限制，避免触发风控

## 执行检查清单

**每执行 5 条命令后，必须检查**：

```
□ 是否所有命令都是串行执行的？
□ 每条命令之间是否等待了 40-55 秒？
□ 是否检查了 REPLIED_CIDS？
□ 是否检查了作者冷却期（≥ 7 天）？
□ 是否使用了 ≥ 2 种不同风格？
□ 是否违反了内容禁令？
□ 获取评论时是否使用了 --page 1 限制？
```

**如果发现违规**：立即停止本轮，记录原因到执行报告。
